Задача 1. Вычисление суммы чисел

Описание задачи

Необходимо написать три программы для вычисления суммы всех чисел от 1 до 10000000000000. Каждая программа должна использовать разные подходы параллелизма: threading, multiprocessing и async.

Для ускорения вычислений используется формула суммы арифметической прогрессии:

сумма от start до end = (start + end) * (end - start + 1) // 2

Диапазон разбивается на несколько частей для параллельной обработки.

Реализация threading

Используется модуль threading для создания потоков. Диапазон разбивается на 8 частей, каждая обрабатывается отдельным потоком. Результаты собираются в общий список с использованием threading.Lock для синхронизации.

Код находится в файле task_1/threading_solution/main.py.

Реализация multiprocessing

Используется модуль multiprocessing с multiprocessing.Pool. Диапазон разбивается на количество частей, равное количеству ядер процессора. Каждый процесс обрабатывает свою часть и возвращает результат.

Код находится в файле task_1/multiprocessing_solution/main.py.

Реализация async

Используется модуль asyncio с asyncio.gather(). Диапазон разбивается на 8 частей, каждая обрабатывается асинхронной задачей. Важно отметить, что async не даёт преимущества для CPU-bound задач, так как вычисления блокируют событийный цикл.

Код находится в файле task_1/async_solution/main.py.

Результаты времени выполнения

Подход Время выполнения
Threading ~0.001 секунд
Multiprocessing ~0.1 секунд
Async ~0.001 секунд

Примечание: Время зависит от количества ядер процессора и нагрузки на систему.

Вывод по задаче 1

Для данной CPU-bound задачи threading и async показывают похожие результаты, так как вычисления выполняются очень быстро благодаря формуле арифметической прогрессии. Multiprocessing имеет большие накладные расходы на создание процессов, поэтому для такой простой задачи он работает медленнее. Однако для более сложных вычислений multiprocessing был бы оптимальным выбором.
