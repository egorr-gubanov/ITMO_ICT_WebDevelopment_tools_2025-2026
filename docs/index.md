# Лабораторная работа 2. Потоки. Процессы. Асинхронность.

## Цель работы

Понять отличия между потоками и процессами, а также изучить асинхронность в Python. Работа с потоками, процессами и асинхронностью помогает развить навыки создания эффективных и быстродействующих программ, что важно для работы с большими объемами данных и выполнения вычислений.

## Краткое описание подходов

### Threading (Потоки)

Потоки позволяют выполнять несколько задач concurrently в рамках одного процесса. В Python потоки управляются операционной системой и подходят для I/O-bound задач, где программа много времени ожидает ввода-вывода. Однако из-за GIL (Global Interpreter Lock) потоки в Python не могут использовать несколько ядер процессора для CPU-bound задач.

### Multiprocessing (Процессы)

Мультипроцессинг создает отдельные процессы, каждый из которых имеет свой интерпретатор Python и свою память. Это позволяет обойти ограничение GIL и использовать несколько ядер процессора. Подходит для CPU-bound задач, где требуется много вычислений.

### Async (Асинхронность)

Асинхронное программирование использует один поток, но позволяет эффективно переключаться между задачами во время ожидания операций ввода-вывода. Идеально подходит для I/O-bound задач с большим количеством сетевых запросов или операций с файлами.
