Задача 2. Параллельный парсинг веб-страниц

Описание задачи

Необходимо написать три программы для параллельного парсинга веб-страниц с сохранением данных в базу данных SQLite. Каждая программа должна загружать HTML-страницу, извлекать заголовок из тега <title> и сохранять результат в базу данных.

База данных

Используется SQLite с таблицей pages:

CREATE TABLE pages (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    url TEXT NOT NULL,
    title TEXT,
    created_at TEXT NOT NULL
);

База данных создаётся автоматически при запуске каждой программы.

Реализация threading

Используется модуль threading для создания потоков. Для HTTP-запросов используется библиотека requests, для парсинга HTML - BeautifulSoup. Каждый поток открывает отдельное подключение к SQLite.

Код находится в файле task_2/threading_solution/main.py.

Реализация multiprocessing

Используется multiprocessing.Pool для распределения задач по процессам. Для HTTP-запросов используется requests, для парсинга - BeautifulSoup. Каждый процесс открывает отдельное подключение к SQLite.

Код находится в файле task_2/multiprocessing_solution/main.py.

Реализация async

Используется asyncio с библиотеками aiohttp для асинхронных HTTP-запросов и aiosqlite для работы с SQLite. Задачи выполняются через asyncio.gather().

Код находится в файле task_2/async_solution/main.py.

Результаты времени выполнения

Подход Время выполнения
Threading ~2-5 секунд
Multiprocessing ~2-5 секунд
Async ~1-3 секунд

Примечание: Время зависит от скорости сети и доступности серверов.

Вывод по задаче 2

Для I/O-bound задачи парсинга веб-страниц все три подхода показывают хорошие результаты. Async показывает лучшее время, так как не создаёт дополнительные потоки или процессы и эффективно переключается между задачами во время ожидания ответов от серверов. Threading также хорошо подходит для таких задач, несмотря на GIL, так как во время ожидания I/O GIL освобождается. Multiprocessing работает корректно, но имеет большие накладные расходы.
